# DBT 技能教学平台 — 管理员使用指南

## 一、你的角色

作为**管理员（admin）**，你拥有系统的最高权限，可以：

- 创建和管理所有用户账号（学生、报告查看者、其他管理员）
- 生成和管理邀请码
- 查看、修改、删除系统中的所有数据
- 上传和管理知识库文档（教学素材）
- 导出学生原始数据（JSON / CSV）
- 查看学生个体报告和导出 PDF
- 查看操作日志和报告访问日志
- 管理教学会话、测试记录、心情记录、成就、风险事件等全部数据

> 管理员账号拥有最高权限，请妥善保管密码，不要与他人共享。

---

## 二、登录后台管理

### 2.1 访问地址

在浏览器地址栏输入：

```
https://<服务器地址>:10443/admin/
```

### 2.2 登录

1. 输入管理员账号和密码
2. 登录成功后进入 Django 管理后台首页

> 只有角色为 `admin` 的账号才能访问后台管理页面。学生和报告查看者无法进入。

### 2.3 管理员账号的创建

**首个管理员账号**由系统部署时通过以下命令创建：

```bash
docker compose exec web python manage.py createsuperuser
```

**后续管理员账号**可在后台管理页面中创建（参见第三章）。

## 三、用户管理

在后台首页点击 **"Users"**（用户），进入用户列表页面。

### 3.1 用户列表

用户列表展示以下信息：

| 列 | 说明 |
|------|------|
| 用户名 | 登录账号名 |
| 角色（role） | `student`（学生）、`report_viewer`（报告查看者）、`admin`（管理员） |
| 邀请码 | 注册时使用的邀请码（仅学生） |
| 已完成问卷 | 是否完成了个人问卷 |
| 激活状态 | 账号是否启用 |
| 最后登录 | 最近一次登录时间 |

**筛选**：可按角色、问卷完成状态、激活状态进行筛选。

**搜索**：可按用户名或邀请码搜索。

### 3.2 创建新用户

1. 在用户列表页点击右上角 **"增加 Users"** 按钮
2. 填写以下信息：
   - **用户名**（必填）
   - **密码**（必填）
   - **Role**：选择 `student`、`report_viewer` 或 `admin`
   - **Invite code**：如为学生角色，可填写邀请码（选填）
   - **Profile completed**：勾选表示问卷已完成（通常不勾选）
3. 其他字段（邮箱、姓名等）为选填
4. 点击 **"保存"**

> **注意**：选择 `admin` 角色时，系统会自动设置该用户为"工作人员"和"超级用户"状态。选择其他角色时，这两个状态会自动清除。

### 3.3 查看用户详情

点击任意用户名进入用户详情页。详情页除了基本信息外，还包括**关联数据区**，根据用户角色动态显示不同的数据块：

**所有用户都会显示：**

| 数据块 | 内容 |
|------|------|
| 个人档案 | 性别、年龄、年级、爱好标签、困扰标签 |
| 教学会话 | 该用户的所有教学记录（状态、技能、时间） |
| 测试记录 | 该用户的所有测试（次数、正确数、通过状态） |
| 心情记录 | 该用户的所有心情记录（分值、场景、时间） |
| 风险事件 | 该用户的风险事件（检测来源、触发文本摘要、时间） |
| 成就 | 该用户已解锁的成就列表 |
| 知识检索日志 | 该用户与 AI 交互时的知识库检索记录 |

**仅 admin 角色显示：** 管理员操作日志

**仅 report_viewer 角色显示：** 学生授权记录、报告访问日志

**仅 student 角色显示：** 被授权查看者记录、报告被访问日志

> 所有关联数据块为**只读展示**，不能直接在用户详情页中添加或修改。如需修改，请前往对应模块。

### 3.4 修改用户信息

1. 进入用户详情页
2. 修改对应字段
3. 点击 **"保存"**

**常见操作：**

- **重置密码**：点击密码字段下方的"更改密码"链接，或使用详情页下方的密码修改表单
- **禁用账号**：取消"激活状态"勾选后保存
- **更改角色**：修改 Role 字段后保存（系统会自动调整权限标记）

## 四、邀请码管理

在后台首页点击 **"Invite codes"**（邀请码）。

### 4.1 邀请码列表

| 列 | 说明 |
|------|------|
| 邀请码 | 8 位字母数字组合 |
| 状态 | 激活 / 已使用 / 已停用 |
| 创建者 | 生成该邀请码的管理员 |
| 使用者 | 使用该邀请码注册的学生 |
| 创建时间 | 邀请码生成时间 |
| 使用时间 | 邀请码被使用的时间 |

### 4.2 批量生成邀请码

1. 在邀请码列表页，勾选任意数量的邀请码（或全选）
2. 在顶部的 **"动作"** 下拉菜单中选择 **"生成一批邀请码"**
3. 点击 **"执行"**
4. 系统自动生成 **10 个新的 8 位邀请码**，状态为"激活"

### 4.3 单个创建邀请码

1. 点击右上角 **"增加 Invite codes"**
2. 输入邀请码字符串（自定义或随机生成）
3. 状态选择"激活"
4. 点击 **"保存"**

### 4.4 启用 / 停用邀请码

1. 勾选需要操作的邀请码
2. 在 **"动作"** 菜单中选择：
   - **"启用所选邀请码"** — 将已停用的邀请码恢复为可用
   - **"停用所选邀请码"** — 使邀请码失效（不影响已使用的）
3. 点击 **"执行"**

> **注意**：已使用的邀请码（状态为"已使用"）无法重新激活。

## 五、报告查看者授权（选填）

此功能用于建立报告查看者与学生的对应关系。**当前系统的报告查看者默认可以查看所有学生**，授权关系为可选配置。

在后台首页点击 **"Report viewer assignments"**（报告查看者授权）。

### 5.1 创建授权

1. 点击右上角 **"增加 Report viewer assignments"**
2. 在 **"查看者"** 下拉框中选择一个报告查看者（下拉框仅显示 `report_viewer` 角色用户）
3. 在 **"学生"** 下拉框中选择一个学生（下拉框仅显示 `student` 角色用户）
4. 确保 **"激活"** 已勾选
5. 点击 **"保存"**

### 5.2 批量启用 / 停用授权

1. 勾选需要操作的授权记录
2. 在 **"动作"** 菜单中选择 **"启用所选授权"** 或 **"停用所选授权"**
3. 点击 **"执行"**

## 六、知识库管理

知识库是 AI 教学的核心素材来源。在后台首页点击 **"Knowledge documents"**（知识文档）。

### 6.1 上传新文档

1. 在知识文档列表页，点击右上角 **"增加 Knowledge documents"**
2. 系统会跳转到专用的上传页面
3. 填写以下信息：

| 字段 | 说明 |
|------|------|
| 标题 | 文档名称（如"正念呼吸教学素材"） |
| 模块 | 所属 DBT 技能模块（正念、情绪调节、痛苦耐受、人际效能） |
| 技能 | 具体技能名称（如"观察念头"、"智慧心"） |
| 版本 | 文档版本号（如 1.0） |
| 难度 | 初级 / 中级 / 高级 |
| 适合初学者 | 是否标记为初学者友好 |
| 场景标签 | 逗号分隔的场景关键词（如"焦虑,压力,考前"） |
| 风险标记 | 如有敏感内容需标记（如"自伤,自杀"） |
| 文件 | 选择要上传的文件（支持 .txt、.md、.pdf） |

4. 点击 **"上传"**
5. 系统会自动：
   - 将文件存储到对象存储
   - 将文档状态设为"处理中"
   - 在后台异步执行：文本分割 → 向量嵌入 → 存入 Qdrant 向量数据库

### 6.2 查看文档处理状态

在知识文档列表页可以查看：
- **状态**：处理中 / 已完成 / 失败
- **分段数**：文档被分割成的知识片段数量
- **错误信息**：如果处理失败，显示具体原因

### 6.3 查看文档详情

点击文档标题进入详情页，可以看到：
- 文档基本信息
- 所有知识片段的文本预览和嵌入 ID
- 每个片段的创建时间

> 知识片段为只读展示。如需修改内容，请重新上传文档。

## 七、数据查看与监控

### 7.1 教学会话

在后台首页点击 **"Teaching sessions"**（教学会话）。

**列表信息**：会话 ID、用户、阶段、状态、模块、技能、消息数、开始时间、结束时间

**筛选**：可按状态（进行中/已完成/风险停止/用户终止）、阶段、模块、开始时间筛选

**详情页包含**：
- 基本信息（会话 ID、用户、阶段、状态、时间）
- 教学内容（模块、技能、选择理由、教学摘要、教学计划）
- RAG 上下文（知识检索 ID）
- 情绪记录（教学前后心情 ID）
- **对话消息内联表**：所有对话消息（角色、内容预览、模态、时间），只读

### 7.2 测试记录

在后台首页点击 **"Tests"**（测试）。

**列表信息**：测试 ID、用户、关联会话、第几次、状态、通过、正确数、创建时间

**筛选**：可按状态、通过/未通过、次数筛选

**详情页包含**：
- 基本信息（测试 ID、用户、会话、状态）
- 测试结果（次数、总题数、正确数、通过状态）
- **题目内联表**：所有题目（文本预览、用户答案、正确选项、是否答对），只读
- RAG 上下文

### 7.3 测试题目

在后台首页点击 **"Test questions"**（测试题目），可查看每道题目的详细信息：

| 字段 | 说明 |
|------|------|
| 题目文本 | AI 生成的题目内容 |
| 选项 | 4 个选项列表 |
| 正确选项 | 0-3 对应的正确答案 |
| 用户答案 | 学生选择的答案 |
| 是否答对 | 自动判定结果 |
| 解析 | AI 生成的答案解析 |
| 配图 | 如有情景配图，显示图片和生成模型 |

### 7.4 心情记录

在后台首页点击 **"Mood records"**（心情记录）。

**列表信息**：记录 ID、用户、心情值（1-5）、表情符号、场景（教学前/教学后/测试后/手动）、关联会话、时间

**筛选**：可按场景、心情值、时间筛选

### 7.5 成就管理

在后台首页点击 **"Achievements"**（成就定义）或 **"User achievements"**（用户成就）。

**成就定义列表**：成就名称、描述、图标、启用状态、触发规则

**用户成就列表**：用户、成就名称、解锁时间

> 创建新成就时，需填写触发规则（JSON 格式），系统会根据规则自动检测并解锁。

### 7.6 风险事件

在后台首页点击 **"Risk events"**（风险事件）。

这是需要重点关注的模块。列表显示：

| 列 | 说明 |
|------|------|
| 事件 ID | 唯一标识 |
| 用户 | 触发风险的学生 |
| 检测来源 | `keyword`（关键词匹配）/ `ai`（AI 语义检测）/ `both` |
| 会话是否停止 | 是否导致教学终止 |
| 跟进模式 | 风险处理方式 |
| 触发时间 | 风险检测时间 |

**详情页包含**：
- 风险详情：触发文本、检测来源、是否停止会话
- 处理信息：采取的措施、跟进模式、是否已导出标记

**搜索**：可按用户名、触发文本、处理措施搜索。

> 建议定期检查风险事件列表，关注需要跟进的学生。

### 7.7 用户个人档案

在后台首页点击 **"User profiles"**（用户档案），可查看和修改学生的问卷信息：

- 性别、年龄、年级
- 爱好标签 + 自由填写的其他爱好
- 困扰标签 + 自由填写的其他困扰

## 八、日志与审计

### 8.1 管理员操作日志

在后台首页点击 **"Admin operation logs"**（管理员操作日志）。

记录所有管理员的数据导出操作：

| 列 | 说明 |
|------|------|
| 管理员 | 执行操作的管理员用户名 |
| 操作类型 | 当前主要为 `export_data`（数据导出） |
| 目标类型 | 单个用户 / 批量用户 |
| 目标 ID | 被导出数据的用户 ID |
| 导出格式 | JSON / CSV |
| 导出范围 | 批量导出时的用户数量 |
| 时间 | 操作时间 |

> 此日志为**完全只读**，任何人均无法添加、修改或删除记录，确保审计完整性。

### 8.2 报告访问日志

在后台首页点击 **"Report access logs"**（报告访问日志）。

记录所有报告查看行为：

| 列 | 说明 |
|------|------|
| 查看者 | 访问报告的用户 |
| 查看者角色 | report_viewer / admin |
| 学生 | 被查看的学生 |
| 操作类型 | 查看 / 导出 |
| 报告类型 | 个体报告 |
| 导出格式 | PDF（如为导出操作） |
| 时间 | 访问时间 |

> 此日志同样为**完全只读**。

### 8.3 媒体生成日志

**图片生成日志**（Image generation logs）：记录所有 AI 图片生成的请求、状态、模型和结果。

**语音合成日志**（Audio synthesis logs）：记录所有 TTS 语音合成的请求、文本、语音和结果。

**语音识别日志**（Audio transcription logs）：记录所有语音转文字的请求、识别文本和结果。

以上三个日志均为**只读**（不可添加和修改，可删除）。

## 九、常见使用场景

### 场景 1：为新学生创建账号

1. 进入 **"Invite codes"** → 生成一批邀请码
2. 将其中一个邀请码发给学生
3. 学生使用邀请码在首页自行注册
4. （或）进入 **"Users"** → 直接创建学生账号，设置用户名和密码，角色选 `student`

### 场景 2：为新老师创建账号

1. 进入 **"Users"** → 点击"增加 Users"
2. 填写用户名和密码
3. **Role** 选择 `report_viewer`
4. 保存即可（老师即可查看所有学生报告和导出数据）

### 场景 3：上传新教学素材

1. 进入 **"Knowledge documents"** → 点击"增加 Knowledge documents"
2. 填写标题、模块、技能、难度等信息
3. 选择文件并上传
4. 等待系统自动处理（通常 1-2 分钟）
5. 刷新列表确认状态变为"已完成"

### 场景 4：检查学生整体情况

1. 进入 **"Users"** → 点击目标学生用户名
2. 在用户详情页快速浏览：
   - **教学会话**：了解学习频率和进度
   - **测试记录**：查看正确率和通过情况
   - **心情记录**：关注情绪趋势
   - **风险事件**：确认是否有需要跟进的警示

### 场景 5：排查问题

1. 如学生反馈教学中断：进入 **"Teaching sessions"** → 搜索该学生 → 查看会话状态和对话消息
2. 如学生反馈测试异常：进入 **"Tests"** → 搜索该学生 → 查看测试状态和题目详情
3. 如图片/语音功能异常：进入对应的媒体生成日志 → 查看错误信息
4. 如知识库文档处理失败：进入 **"Knowledge documents"** → 查看文档详情中的错误信息

### 场景 6：数据导出

1. 在浏览器访问报告仪表盘 `https://<服务器地址>:10443/reports/`
2. 在统一页面中，点击学生右侧的 **JSON** 或 **CSV** 链接导出个人数据
3. 或点击页面顶部的 **"导出全部 (JSON)"** 或 **"导出全部 (CSV)"** 按钮批量导出
4. 也可以直接查看报告和导出 PDF

> 注意：原来的独立导出页面（`/export/`）已合并到报告仪表盘。

## 十、常见问题

**Q: 如何创建新的管理员账号？**
A: 两种方式：
1. 后台页面：进入"Users" → 增加新用户 → Role 选择 `admin` → 保存（系统自动设置权限标记）
2. 命令行：`docker compose exec web python manage.py createsuperuser`

**Q: 不小心把管理员角色改成了其他角色怎么办？**
A: 如果还能登录且能看到用户列表（有其他管理员账号），直接改回 `admin` 即可。如果这是唯一的管理员且已被降级无法进入后台，需要通过命令行修复：
```bash
docker compose exec web python manage.py shell -c "
from accounts.models import User
u = User.objects.get(username='你的用户名')
u.role = 'admin'
u.save()
"
```

**Q: 如何重置学生密码？**
A: 进入"Users" → 点击学生用户名 → 在密码字段下方点击"更改密码"链接 → 输入新密码 → 保存。

**Q: 如何禁用某个学生账号？**
A: 进入"Users" → 点击学生用户名 → 取消"激活状态"的勾选 → 保存。该学生将无法登录。

**Q: 知识库文档上传后一直显示"处理中"怎么办？**
A: 首先等待 2-3 分钟，大型文档处理需要时间。如果长时间未完成，检查：
1. Celery worker 是否正常运行：`docker compose ps celery_worker`
2. Qdrant 服务是否正常：`docker compose ps qdrant`
3. 如服务异常，重启相关服务：`docker compose restart celery_worker`

**Q: 如何删除不需要的数据？**
A: 在对应的列表页面，勾选要删除的记录，在"动作"菜单中选择"删除所选"并执行。或在详情页点击"删除"按钮。

> **注意**：操作日志和报告访问日志不可删除（审计用途）。删除操作不可撤销，请在删除前确认。

**Q: 日志记录可以手动添加或修改吗？**
A: 不可以。`AdminOperationLog` 和 `ReportAccessLog` 为完全只读的审计日志，任何人均无法手动添加、修改或删除，确保数据安全性。

**Q: 我可以看到学生的完整对话内容吗？**
A: 可以。两种方式：
1. 后台页面：进入"Teaching sessions" → 点击目标会话 → 在对话消息内联表中查看内容预览 → 点击 ChatMessage 进入详情页查看完整内容

**Q: 如何让学生看不到其他学生的数据？**
A: 系统已通过角色权限控制实现：学生只能看到自己的教学、测试和心情数据。报告查看者和管理员可以看到所有学生的报告。如需更细粒度的报告查看控制，可使用"Report viewer assignments"功能（参见第五章）。

**Q: 后台页面中文显示正常吗？**
A: 是的。后台管理页面的所有字段标签、筛选器、操作按钮均已中文化。

**Q: 如何备份系统数据？**
A: 推荐定期备份 MongoDB 数据库：
```bash
docker compose exec mongodb mongodump --db dbt_platform --out /backup/$(date +%Y%m%d)
```
同时也建议定期通过报告仪表盘的批量导出功能（`/reports/` 页面顶部按钮）导出全部学生数据的 JSON/CSV 作为额外备份。
